# Dub Language — High-Level Design Overview

## Philosophy

- Build from first principles rather than copying existing languages.
- Emphasize clarity, uniqueness, and simplicity.
- Let the language evolve organically through experimentation.
- Keep the AST as the core representation of all code.
- Maintain portability by ensuring the AST is JSON-serializable.

## Architecture Summary

- Source files are plain text (`.dub` planned).
- Parsing is positional and index-driven.
- AST is non-recursive and built from operator/parenthesis indices.
- Execution initially happens in Python for rapid iteration.
- Long-term: bytecode VM or C-based interpreter.

## Development Strategy

- Rotate between projects to avoid burnout and gain cross-disciplinary insights.
- Use supporting tools (BInspected, FigMan, DesignMe) to strengthen different skills.
- Capture design decisions as they evolve.
