# Abstract Syntax Tree (AST)

## Core Properties

- Fully positional: each node stores start/end indices.
- Non-recursive: structure emerges from index ranges.
- JSON-serializable by design.
- Decoupled from runtime execution.

## Node Types (planned)

- Literal
- Identifier
- Binary Operation
- Assignment
- Function Call
- Block
- Parenthesized Expression

## Why JSON Matters

- Acts as a portable intermediate representation.
- Allows multiple runtimes (Python, C, Rust, etc.).
- Enables future bytecode compilation.
- Makes bootstrapping possible.

## Long-Term Vision

- AST → Bytecode → VM
- AST → C → Machine Code
- AST → Dub Interpreter (self-hosting)
